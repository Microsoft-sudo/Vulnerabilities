# SSO Authentication OTP Email Disclosure in Client-Side Code

**ID:** vuln-0006
**Severity:** MEDIUM
**Found:** 2026-03-18 13:29:08 UTC
**Target:** https://accounts.cleartax.in/v2
**Endpoint:** https://accounts.cleartax.in/v2
**Method:** GET
**CWE:** CWE-200
**CVSS:** 4.3

## Description

The ClearTax SSO authentication system (https://accounts.cleartax.in/v2) exposes OTP recipient email addresses in client-side JavaScript code. During OTP verification, the email address to which the OTP was sent is embedded in the page source, allowing any user with access to the page to view the email address. This information disclosure occurs in the OTP verification interface where the system displays "We have sent an OTP to [email]" with the actual email address embedded in the page content.

**Key Findings:**
1. **Email Address Exposure**: OTP recipient email addresses are visible in the DOM during OTP verification
2. **Information Disclosure**: Email addresses used for authentication are exposed to anyone with access to the OTP verification page
3. **Privacy Violation**: Users' email addresses are unnecessarily exposed during the authentication process

## Impact

This vulnerability has several security and privacy implications:

1. **User Privacy Violation**: Exposes users' email addresses which may be considered personal identifiable information (PII)
2. **Account Enumeration**: While not directly enabling account enumeration via the API, it reveals valid email addresses during the authentication flow
3. **Phishing Attack Vector**: Exposed email addresses could be targeted for phishing attacks
4. **Information Disclosure Chain**: Combined with other vulnerabilities, this could facilitate more sophisticated attacks

The exposed email address (`f1d41bc0eb12db930b01844f12c513a1@sentry-v2.clear.in`) appears to be a system-generated email for error tracking, but the vulnerability would apply to any user email address used in the OTP verification flow.

## Technical Analysis

The SSO authentication system at https://accounts.cleartax.in/v2 uses an OTP-based authentication flow. During the OTP verification step, the system displays a message indicating where the OTP was sent. This message includes the actual email address embedded in the page HTML/JavaScript.

**Technical Details:**
- Page element contains text: "We have sent an OTP to [email]"
- Email address is directly embedded in the DOM/text content
- No obfuscation or masking of the email address
- Accessible via simple DOM inspection or JavaScript

**Discovered Email**: `f1d41bc0eb12db930b01844f12c513a1@sentry-v2.clear.in`
- This appears to be a Sentry error tracking email address
- Indicates the vulnerability affects all email addresses in the OTP flow

**Testing Methodology:**
1. Accessed the SSO OTP verification page
2. Used JavaScript to extract text content from the page
3. Identified email addresses using regex pattern matching
4. Confirmed email addresses are embedded in client-side code

## Remediation

1. **Mask Email Addresses in UI**:
   - Display only a portion of the email (e.g., "j***@example.com")
   - Use client-side obfuscation before displaying emails
   - Implement email masking library or custom solution

2. **Server-Side Email Handling**:
   - Store email references as tokens or hashes on the server
   - Only send masked email addresses to the client
   - Use session-based references instead of actual email addresses

3. **Privacy by Design**:
   - Review all user-facing messages for PII exposure
   - Implement consistent email masking across the application
   - Conduct privacy impact assessment for authentication flows

4. **Client-Side Security**:
   - Avoid embedding PII in HTML/JavaScript
   - Use data attributes with encrypted values if email display is necessary
   - Implement proper input/output sanitization

5. **Testing and Validation**:
   - Add automated tests to detect PII exposure in client-side code
   - Conduct regular security reviews of authentication flows
   - Implement data loss prevention monitoring
