# Session Security Vulnerability - ssoAId Cookie Missing HttpOnly Flag

**ID:** vuln-0004
**Severity:** MEDIUM
**Found:** 2026-03-18 13:25:59 UTC
**Target:** https://accounts.cleartax.in/v2
**Endpoint:** /
**Method:** GET
**CWE:** CWE-1004
**CVSS:** 5.4

## Description

The ssoAId analytics cookie on accounts.cleartax.in/v2 is missing the HttpOnly flag, making it accessible to client-side JavaScript. This exposes the analytics identifier to potential theft through XSS attacks, violating security best practices for session-related cookies.

## Impact

Attackers can steal the ssoAId cookie via XSS vulnerabilities, potentially enabling user tracking, session correlation, or analytics data manipulation. While not directly granting authentication access, it compromises user privacy and could be used in combination with other vulnerabilities for more sophisticated attacks.

## Technical Analysis

The ssoAId cookie is set with Secure flag but lacks HttpOnly protection:
- Secure: True (properly implemented)
- HttpOnly: False (security vulnerability)
- SameSite: Not set (should be set to Lax or Strict)
- Path: / (correct)
- Domain: accounts.cleartax.in (correct)

This allows client-side JavaScript to read the cookie value, creating a privacy risk. Analytics identifiers should be treated with the same security considerations as authentication tokens to prevent user tracking and correlation attacks.

## Remediation

1. Add HttpOnly flag to ssoAId cookie in server response headers
2. Set SameSite attribute to Lax or Strict for additional protection
3. Conduct security review of all cookies for proper flag implementation
4. Implement Content Security Policy to mitigate XSS risks
5. Regular security testing of cookie attributes
6. Consider if analytics data needs to be in a cookie or could use alternative storage
